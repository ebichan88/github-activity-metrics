# 評価資料準備時間 ベースライン計測手順

**対象**: SC-004「評価資料作成の準備時間を従来比50%以下に削減する」  
**バージョン**: 1.0.0 | **作成日**: 2026-06-07

---

## 計測方法

### ベースライン計測（ツール導入前）

以下の手順を実行し、所要時間を記録してください。

1. GitHub のリポジトリページを手動で開く
2. 対象期間の PR をフィルタして一覧を確認する
3. 各 PR の件数・マージ状況を手動でスプレッドシートに記入する
4. コミット数・変更行数を手動で確認して記入する
5. レビュー数・コメント数を手動で確認して記入する
6. 評価資料（スプレッドシート等）を完成させる

**計測単位**: 担当者1名あたりの上記手順の所要時間（分）

| 計測回 | 担当者数 | 対象期間 | 所要時間（分） | 備考 |
|--------|----------|----------|----------------|------|
| 1      |          |          |                |      |
| 2      |          |          |                |      |
| 平均   |          |          |                |      |

---

### ツール導入後の計測

1. `./scripts/export-activity-json.sh` で JSON を収集する（収集時間は含めない）
2. JSON ファイルをダッシュボードで開く
3. KPI カード・担当者テーブルの内容を評価資料に転記する
4. 評価資料を完成させる

**計測単位**: 担当者1名あたりの転記作業の所要時間（分）

| 計測回 | 担当者数 | 対象期間 | 所要時間（分） | 備考 |
|--------|----------|----------|----------------|------|
| 1      |          |          |                |      |
| 2      |          |          |                |      |
| 平均   |          |          |                |      |

## 判定基準

```
削減率 = 1 - (ツール使用後の平均時間 / ベースライン平均時間)
```

- **合格**: 削減率 ≥ 50%（例: ベースライン30分 → ツール使用後15分以下）
- **不合格**: 削減率 < 50%

## 計測支援スクリプト

詳細は `scripts/measure-report-prep-time.sh` を参照してください。

```bash
./scripts/measure-report-prep-time.sh start  # タイマー開始
./scripts/measure-report-prep-time.sh stop   # タイマー停止・結果記録
```
