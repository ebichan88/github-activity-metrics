# 手動テスト観点: GitHub活動実績ローカルダッシュボード

**バージョン**: 1.0.0 | **作成日**: 2026-06-07

## 1. 収集系テスト（collector）

### MT-01: 正常系収集

- [ ] gh CLI でログイン済みの状態で `./scripts/export-activity-json.sh` を引数なし実行 → Usage が表示される
- [ ] `--from`/`--to`/`--repos`/`--contributors`/`--out` をすべて指定して実行 → JSON ファイルが生成される
- [ ] 生成された JSON を `cat | python3 -m json.tool` でパースできる
- [ ] 出力 JSON に `datasetVersion: "1.0.0"` が含まれる

### MT-02: 期間・複数リポジトリ指定

- [ ] 複数リポジトリをカンマ区切りで指定して実行 → 全リポジトリの PR/コミットが集計される
- [ ] 期間を変えて2回実行 → それぞれ異なるファイルに出力できる
- [ ] 短い期間（1日）で実行 → 正常終了する

### MT-03: エッジケース

- [ ] PR作成数0件の担当者を含む → `derived` の各指標が `null` になる
- [ ] アクセス権なしリポジトリを含む → 警告付きで他リポジトリの集計は完了する
- [ ] `--contributors` に存在しない GitHub ユーザーを指定 → 空データ（0件）で正常終了する

### MT-04: エラー系

- [ ] gh CLI 未ログイン状態で実行 → 認証エラーメッセージが表示される
- [ ] `pnpm` 未インストール環境で実行 → pnpm インストール要求メッセージが表示される
- [ ] 不正な日付形式（`--from 2026/05/01`）で実行 → バリデーションエラーが表示される

## 2. フロントエンド系テスト

### MT-05: JSON 読み込み

- [ ] `samples/activity-dataset.sample.json` を UI からファイル選択 → KPI カードと担当者テーブルが表示される
- [ ] 表示まで 3 秒以内に完了する
- [ ] 期間・生成日時がヘッダーに表示される

### MT-06: 表示内容確認

- [ ] KPI カード: PR作成数・PRマージ数・コミット数・レビュー数・追加行・削除行が表示される
- [ ] 担当者テーブル: ログイン・各指標の列が揃っている
- [ ] PR作成数0件の担当者の `mergeRate`/`reviewRate` が `-` と表示される
- [ ] 警告があるデータセットで警告バナーが表示される

### MT-07: Local First 確認

- [ ] オフライン状態（ネットワーク無効）でも保存済み JSON を読み込んで表示できる
- [ ] ページリロード後にファイル選択から再度読み込める

### MT-08: エラー表示

- [ ] 不正 JSON ファイルを選択 → エラーバナーと「別のファイルを選択」ボタンが表示される
- [ ] 未サポートバージョンの JSON → バージョンエラーメッセージが表示される
- [ ] エラー後に別の正しい JSON を選択 → 正常に表示される

### MT-09: FR-010 ログイン導線非提供

- [ ] UI 上にログインボタン・OAuth認証・PAT入力フォームが存在しない
- [ ] UI に GitHub 認証関連の UI 要素がない

### MT-10: GitHub Pages 確認

- [ ] Pages URL でアクセス → ファイル選択画面が表示される
- [ ] Pages 環境でサンプル JSON を読み込み → 正常にダッシュボードが表示される

## 3. 非機能テスト

### MT-11: パフォーマンス

- [ ] 100 リポジトリ・31 日範囲での収集が 15 分以内に完了する（SC-001）
- [ ] 読み込み後 3 秒以内に主要カード表示（SC-002）

### MT-12: SC-004 評価資料準備時間

- [ ] `scripts/measure-report-prep-time.sh` を使ってベースライン計測を実施する（詳細は `checklists/reporting-time-benchmark.md` 参照）
